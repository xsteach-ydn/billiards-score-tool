# 项目简报

## 产品定位

Billiards Score Tool 是一个轻量桌球记分与结算工具。

主要使用场景是一小群朋友在一次活动中连续打多场，每场记录所有人的分数变化，最后查看排名并按需要把积分换算成金额。

## 目标用户

- 经常 3 到 5 人一起打桌球的休闲玩家。
- 偶尔扩展到最多 7 人的朋友局。
- 希望在手机上快速记分、查看历史、继续活动、复制结果到微信的用户。

## 核心问题

手工记分容易出错，因为每场比赛分数必须相加为 `0`，活动总分也必须保持平衡。活动结束后，玩家还可能需要按积分换算金额并分享结果。

这个工具应让录分更快、阻止非法场次、保留历史明细，并让结果分享足够顺手。

## 当前范围

- 使用默认日期时间辅助创建活动名称。
- 支持 3 到 7 名玩家。
- 复用最近玩家姓名。
- 记录每一场的时间和每位玩家得分。
- 要求每一场得分合计为 `0`。
- 展示活动总分、排名和金额换算。
- 可继续进行中的历史活动。
- 可查看已结束活动明细。
- 通过导入/导出备份和恢复全部记录。
- 分享单个活动结果。

## 产品规则

- 默认金额换算为 `1 分 = 5 元`。
- 导入/导出是全局备份/恢复，不是单活动分享。
- 分享是单活动结果分享。
- 活动时长根据时间戳计算。
- 场次录入时间在草稿为空时自动更新；用户手动修改后停止自动覆盖。

## 当前原型暂不包含

- 用户账号。
- 云端同步。
- 多设备协作。
- 支付收款。
- 锦标赛/淘汰赛赛制。
- 高级统计分析。

这些能力以后可通过 `apps/api` 和 `packages/shared` 扩展。

## 交互原则

- 首屏就是活动记录，不做营销页。
- 高频动作要足够直接：继续活动、新建活动、保存本场、结算、分享。
- 数字信息要易扫读。
- 通过控件和校验预防错误，尤其是零和场次。
- 界面应实用、安静、高效，不做过度装饰。
