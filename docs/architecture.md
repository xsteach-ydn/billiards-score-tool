# 架构说明

## 当前状态

项目目前是前端优先原型。

```text
apps/web
  React + Vite 应用
  localStorage 持久化
  移动端优先 UI

apps/api
  预留给服务端

packages/shared
  预留给共享领域逻辑和接口契约
```

## 仓库结构

- `apps/web`：当前 Web 应用。
- `apps/api`：未来服务端应用。
- `packages/shared`：未来共享包，用于计分规则、数据类型、校验 schema、导入导出 schema 和 API 契约。
- `docs`：给人和 AI 工具共同使用的稳定项目资料。

## 前端职责

当前 Web 应用负责：

- 活动创建和编辑。
- 场次分数录入。
- 零和校验。
- 排名和总分计算。
- 金额换算展示。
- 结算建议。
- 导入/导出文本生成和解析。
- 适合微信复制的活动结果分享文本。
- 浏览器本地持久化。

## 未来 API 职责

开始服务端工作时，代码放在 `apps/api`。

可能职责包括：

- 用户身份和会话管理。
- 云端持久化和备份。
- 多设备同步。
- 活动结果分享链接。
- 进行中活动的多人协作。
- 服务端导入/导出校验。

## 未来共享包职责

当某些逻辑需要被前端和服务端共同使用时，放到 `packages/shared`。

适合抽取的内容：

- 活动、玩家、场次、分数的数据类型。
- 零和校验。
- 总分和排名计算。
- 金额换算。
- 结算建议算法。
- 导入/导出 schema 和迁移。
- API 请求/响应契约。

## 当前数据流

1. Web 应用从 `localStorage` 读取活动记录。
2. 应用打开期间，React state 是界面状态来源。
3. 状态变化后序列化回 `localStorage`。
4. 导入可以替换或合并活动记录。
5. 导出会序列化全部活动记录用于备份。
6. 分享会把单个活动格式化成可复制文本。

## 演进方向

下一步架构演进建议：把 `apps/web/src/App.jsx` 里的纯领域逻辑抽到 `packages/shared`。

推荐抽取顺序：

1. 分数求和和空分数辅助函数。
2. 活动总分和排名。
3. 金额换算。
4. 结算建议。
5. 导入/导出 schema 校验。

之后 `apps/api` 可以复用同一套规则，而不是重新实现。

## AI 协作注意事项

- 不要把服务端代码塞进 `apps/web`。
- `packages/shared` 建立后，不要在多个应用中重复实现领域规则。
- 架构边界变化时，同步更新本文档。
- 重要架构决策记录到 `docs/decisions`。
