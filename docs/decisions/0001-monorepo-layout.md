# 0001 使用 Monorepo 结构

## 状态

已接受

## 日期

2026-06-15

## 背景

项目最初是桌球记分前端原型，但后续预计会增加服务端能力和共享业务规则。

如果所有内容都放在一个扁平前端目录里，后续拆服务会更困难，也容易重复实现计分逻辑。

## 决策

采用小型 monorepo 结构：

```text
apps/web
apps/api
packages/shared
docs
```

当前 React + Vite 应用放在 `apps/web`。

`apps/api` 预留给未来服务端代码。

`packages/shared` 预留给前后端复用的计分规则、校验、schema 和 API 契约。

## 影响

- 根目录脚本委托给具体应用脚本。
- AI 工具新增代码前应先确认仓库边界。
- 共享领域逻辑应逐步从 `apps/web` 抽出。
- 文档需要保持更新，方便多个 AI 工具安全协作。
